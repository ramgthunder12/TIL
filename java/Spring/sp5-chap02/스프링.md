# 스프링

핵심 : 객체 생성, 초기화

## 빈

스프링에서 생성되는 객체를  Bean 빈 객체 라고 부른다.

### @Bean 애노테이션
 
@Bean 애노테이션을 메서드에 붙이면 해당 메서드가 생성한 객체를 스르링이 관리하는 빈 객체로 등록한다.

빈 애노테이션을 붙인 메소드의 명은 빈 객체를 구분할 때 사용한다.

### BeanFacotry, AppicationContext, 컨테이너

- BeanFactory : 스프링 IoC 컨테이너의 최상위 인터페이스. 빈의 생성, 조회, 라이프사이클 관리의 기본 기능을 정의한 인터페이스다.  

- ApplicationContext : BeanFactory를 상속 받은 인터페이스로, 빈 관리 기능 + 메시지 소스, 이벤트, AOP, 환경설정, 리소스 로딩 등 엔터프라이즈 기능을 제공하는 인터페이스다.


- 스프링 컨테이너 : 빈 이름과 실제 객체의 연결 정보, 실제 객체의 생성, 초기화, 의존 주입, 객체 관리를 위한 다양한 기능 제공, BeanFactory와 ApplicationContext 등을 구현한 객체를 의미한다.
