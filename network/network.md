# Network

네트워크에 관한 내용 정리

## TCP, UDP

TCP : 패킷을 목적지에 전달하고 잘 전달되었는지 확인하는 방법 이다.  
UDP : 패킷을 목적지에 전달만 한다. 빠르지만 패킷손실이 있을 수 있다. 

## Socket

데이터를 연결하는 중간 연결고리(전구 소켓) 장치, 네트워크에 연결 하려면 소켓에 연결 해야합니다.   
네트워크와 연결을 담당하는 객체입니다. 

### Server Socket(Listener)
데이터를 받는곳에서만 사용하는 추가적인 장치입니다.
서버의 역할인 곳에서만 필요합니다.  
serversocket은 서버 내에서, 외부에서 들어오는 다른 소켓들이 연결되기를 기다리는 역할을 합니다. 따라서 리스너라고 부릅니다.

ServerSocket은 accept()메소드를 사용해 상대방과 연결 정보를 그대로 가진 socket객체를 만들어 냅니다.  
ServerSocket 생성시 포트번호 필요합니다.  


## Protocol

약속, 어떤 식으로 메시지를 주고받거나 어떤 구조에 맞춰서 데이터를 보내고 받는 것을 의미합니다.

## HTTP

Application 계층 프로토콜입니다.  
기본적으로 80번 포트 사용

## HTTPS

Http + SSL/TLS로 암호화  
SSL/TLS 인증서 필요합니다.
TLS를 사용해 암호화, 인증, 무결성을 보장하는 보안 버전의 HTTP입니다.
443포트 이용
